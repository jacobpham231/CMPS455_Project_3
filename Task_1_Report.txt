Task 1.1 - Implementation Overview
This task implements a single-core CPU scheduler simulator using Java threads. The simulator supports four scheduling algorithms:
1. FCFS (First-Come, First-Served)
2. RR (Round Robin)
3. NSJF (Non-Preemptive Shortest Job First)
4. PSJF (Preemptive Shortest Job First)

The implementation uses one shared ready queue protected by a lock, one dispatcher thread, and one thread per task. Each task thread represents a process with total CPU burst time B. The dispatcher selects a task according to the chosen algorithm and grants CPU time through a separate CPU function.

Task 1.2 - Thread and Scheduler Design
1. Task threads
- Each task has: taskId, totalBurst, remainingBurst, and arrivalTick.
- Each task runs a common thread function that waits until the dispatcher grants cycles.
- Each loop iteration in the task simulates one CPU cycle.
- After each granted slice, the task notifies the dispatcher.

2. Dispatcher
- The dispatcher repeatedly:
  a) Enqueues newly arrived tasks
  b) Selects the next task from the ready queue
  c) Grants cycles according to scheduling policy
  d) Waits for completion of the granted slice
  e) Requeues unfinished tasks when required
- The simulation ends when all tasks complete.

3. CPU function separation
- CPU execution is represented by a separate method in the dispatcher, satisfying the requirement that CPU logic is separated from dispatcher decision logic.

Task 1.3 - Scheduling Behavior Implemented
1. FCFS
- Selects the earliest task in queue order.
- Runs selected task until completion.

2. RR
- Selects in queue order.
- Runs task for min(quantum, remainingBurst).
- Requeues unfinished task at queue tail.

3. NSJF
- Selects task with smallest remaining burst from the ready queue.
- Runs selected task to completion (non-preemptive).

4. PSJF
- Tasks can arrive after simulation start.
- Dispatcher reevaluates selection every tick by granting one cycle at a time.
- If a shorter remaining task arrives, selection naturally preempts on the next scheduling decision.

Task 1.4 - Synchronization and Safety

Task 1.7 - Which Algorithm Was Fastest?
Result: tie.
All four algorithms completed in 113 ticks for this workload in this simulator.

Explanation:
The simulator accounts for CPU burst execution time but does not model explicit context-switch overhead. Since total burst work is fixed (18 + 7 + 25 + 42 + 21 = 113), total completion time in ticks is identical across these four policies under this timing model.
